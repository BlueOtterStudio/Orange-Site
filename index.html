How to use these pages in your app

Publish this folder as a GitHub repository and enable GitHub Pages; then use the published URLs in your app for links to Privacy, Terms, FAQ, Contact and Support pages.

  1. Create a GitHub repository and push the contents of this folder to the repository root (main branch).
  2. In GitHub: go to Settings → Pages, select the branch (e.g. main) and root (/) as the source, then Save. Wait a few minutes for the site to publish.
  3. Add the published URLs into your app. Example URLs (replace placeholders):
    • Privacy: https://your-username.github.io/your-repo/privacy.html
    • Terms: https://your-username.github.io/your-repo/terms.html
    • FAQ: https://your-username.github.io/your-repo/faq.html
    • Contact: https://your-username.github.io/your-repo/contact.html
    • Support: https://your-username.github.io/your-repo/support.html

Notes:

  • If you prefer a separate branch, you can publish from a gh-pages branch instead of main.
  • Use a custom domain by adding a CNAME file with your domain and configuring DNS records; see GitHub Pages docs for details.
  • After publishing, test each URL and update the links in your app to point to the stable published addresses.
